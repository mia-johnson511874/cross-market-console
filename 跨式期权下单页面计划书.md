# 跨式/宽跨式期权下单页面 — 实施计划书

> 依据：`跨市期权下单页面对话1.txt`、`跨式期权下单对话页面2.txt`（多头宽跨 Strangle → 跨式 Straddle 的盈亏平衡点收窄方案）
> 目标项目：cross-market-console（React 19 + TypeScript + Vite 前端，FastAPI 后端）

---

## 一、需求理解

两份对话文档的核心内容：用户持有**多头宽跨式（Long Strangle：买入 CALL + 买入 PUT，行权价不同）**，关注：

1. **盈亏平衡点计算**
   - 下方平衡点 = PUT 行权价 − 每股总权利金
   - 上方平衡点 = CALL 行权价 + 每股总权利金
   - 区间宽度 = 2 × 每股成本 − 行权价差
2. **缩小盈亏平衡点的 4 类方案**（页面需内置该"方案对比"能力）
   - 方案1：改同行权价跨式 Straddle（价差→0，效果最强，权利金更贵）
   - 方案2：缩小 CALL/PUT 行权价差（保留宽跨结构）
   - 方案3：降低总权利金（更深虚值 / 更短到期）
   - 方案4：卖出外侧期权收权利金（多宽跨 + 空窄跨，收益封顶）
3. **关键指标**：净权利金支出、最大亏损、上下平衡点、区间宽度、希腊字母（Delta/Gamma/Theta/Vega）
4. **方案对比汇总表**：行权组合 / 每股成本 / 盈亏区间 / 区间宽度 / 特点

因此新页面 = **跨式/宽跨式下单页**：选品种与到期月 → 选 CALL/PUT 行权价（同行权即跨式、不同行权即宽跨）→ 实时权利金 → 自动计算平衡点与最大亏损 → 一键生成"收窄平衡点"备选方案并对比 → 模拟下单。

### 3.2 数据流（沿用系统现有连接方式）

```
StraddleOrderPage
  ├─ 标的实时价：fetchGridSnapshot(gridId)  ←  /api/grid-products/{id}/snapshot（东财→YF→akshare→模拟）
  ├─ 期权链：    fetchOptionChainByProduct  ←  /api/option-products/{id}/chain（akshare，失败→模拟权利金）
```

- 链上取到 CALL/PUT 最新价 → 自动填充权利金输入框（可手改，与现有页面一致）

### 3.3 计算引擎 `straddleEngine.ts`（核心）

```ts
// 输入：callStrike, putStrike, callPremium, putPremium, contracts, pointValue, 标的价格, 到期天数, 波动率(估计)
// 输出：
{
  structure: 'straddle' | 'strangle',      // 同行权=跨式，否则宽跨
  debitPerShare,                            // 每股净权利金 = call + put
  netDebit,                                 // 总净支出 = debitPerShare × 数量 × pointValue换算
  maxLoss,                                  // 最大亏损 = 总净支出（多头）
  lowerBreakeven,                           // putStrike − debitPerShare
  upperBreakeven,                           // callStrike + debitPerShare
  intervalWidth,                            // upper − lower
  maxProfit: '无限',
  greeks: { delta, gamma, theta, vega },    // Black-Scholes 欧式定价（ETF期权），IV 由权利金反推或用默认 25%
  payoffCurve: Array<{ price, pnl }>,       // 到期损益曲线采样点（SVG 用）
}
```

备选方案生成器（对应需求文档 4 方案，用真实链上权利金测算，缺价时用 BS 估算）：

1. **转平值跨式**：CALL/PUT 都移到最接近现价的行权价
2. **行权价差减半**：CALL/PUT 各向内靠一档（链上实际行权档位）
3. **降权利金**：CALL/PUT 各向外移一档更深虚值（或换更近到期月）
4. **卖外侧收权利金**：保留原组合 + 卖出更低行权 PUT、更高行权 CALL，重算净支出与平衡点（同时标注"收益封顶"）

每个方案输出：新平衡点区间、区间宽度、净支出、最大亏损变化、与原组合对比的**汇总表**（仿对话2的对比表）。

## 五、边界与说明

- **下单为模拟下单**：与现有两个下单页一致，只写本地日志，不接真实柜台
- **商品期权/港股指数期权**：链接口本身无合约数据（后端注明需 CTP/OMID），这些品种进入"手动权利金 + 静态兜底"模式，功能照常可用
- **希腊字母**：现有页面是伪算法，本页升级为 BS 模型真算（欧式、无股息，ETF 期权适用）；IV 优先从链上价格反推，失败用 25% 默认
- **对话中的 SNXX 美股案例**：仅作为算法验证样例，页面品种仍以系统内 11 个期权品种为准
